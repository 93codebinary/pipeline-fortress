# OWASP Juice Shop — PipelineFortress Code of Conduct

## Overview

This project adheres to the [OWASP Code of Conduct](https://policy.owasp.org/operational/code-of-conduct.html). All community members, contributors, pipeline engineers, and security researchers participating in this project are expected to follow these guidelines to maintain a respectful, inclusive, and professional environment.

This project is maintained by 93codebinary as part of the IIT Roorkee × Futurense PG Certificate in AI/GenAI Powered Cybersecurity (Capstone CAP-DSE-3W).

---

## OWASP Code of Conduct

For comprehensive information on expected behavior, community standards, and enforcement procedures, please refer to the official [OWASP Code of Conduct policy](https://policy.owasp.org/operational/code-of-conduct.html).

---

## Responsible Use — PipelineFortress Specific

Because this project wraps a **deliberately vulnerable application** (OWASP Juice Shop) in a live DevSecOps pipeline, all participants must additionally follow these rules:

- All scanning, exploitation, and testing must be confined to the lab or sandbox environment. **No real-world targets.**
- Any use of GenAI or LLMs must be disclosed — which model, what it was used for, and how outputs were verified.
- Never paste real credentials, secrets, customer data, or copyrighted content into any LLM.
- All scan findings, logs, and evidence handled in this project remain confidential within the team.
- Pipeline artifacts and security reports must not be shared publicly without review.

## Scope

This Code of Conduct applies to all spaces associated with this project, including:

- GitHub repository, issues, and pull requests (`93codebinary/pipeline-fortress`)
- CI/CD pipeline runs, security scan results, and artifact outputs
- Community forums and chat channels (Slack, Gitter, Reddit etc.)
- Official project events, presentations, and communications
- Any spaces where individuals represent this project or OWASP Juice Shop
